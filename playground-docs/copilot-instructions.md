# Copilot Instructions for vue-sdk-demo

This repository contains a **Vue.js demo application** that demonstrates how to use the Vue SDK.

Copilot should follow these guidelines when generating or modifying code in this project.

## Project Goals
- Demonstrate the usage of the Vue SDK in a clear and simple way.
- Help developers understand how authentication and SDK features work in a Vue application.

## Code Style and Complexity
- Keep the code **simple and easy to understand**, especially for developers who are new to Vue.js.
- Avoid unnecessary abstractions, patterns, or complex logic.
- Prefer **clear and readable code** over clever implementations.
- Use modern Vue practices such as **Composition API and `<script setup>`** when appropriate.

## Project Structure
- Keep the codebase **minimal and well organized**.
- Components should be placed inside the `components` directory.
- Pages/views should be placed inside the `views` directory.
- Keep authentication or SDK-related logic separated into reusable utilities or composables.

## UI and Design
- The UI should be **minimal, clean, and easy to understand**.
- Avoid complex styling or large color variations.
- Focus on **clarity and usability rather than visual complexity**.
- Use Tailwind CSS for styling to keep the code concise and maintainable.
- Support both light and dark modes properly.

## Documentation
- Add **clear comments when implementing SDK-related functionality**.
- Include short explanations where necessary so developers can easily understand the flow.

## Demo Behavior
The application should clearly demonstrate:

- Login
- Logout
- Retrieving user information
- Calling protected APIs

All flows should be **easy to locate and understand** in the codebase.
